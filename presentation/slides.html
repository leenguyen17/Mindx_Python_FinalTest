Technical Implementation

  • Pandas boolean masks — chain filters efficiently on 8,807 rows without database overhead
  • Pre-computed filter options — genres and countries are exploded from comma-separated fields at startup
  • Debounced search — 300ms delay prevents excessive API calls while typing
  • Destroy-before-create pattern — prevents Chart.js "canvas already in use" errors on re-render
  • NaN handling — fillna + where(pd.notnull) for clean JSON serialization
  • Modular JS architecture — api.js / components.js / charts.js / app.js separation of concerns
  • Staggered card animations — CSS animation-delay for polished load experience
  • fpdf2 PDF generation — landscape table with Netflix-red header, filter summary, up to 500 rows

Challenges & Learnings

  • Data cleaning — handling missing directors (2,634), cast (825), and country (831) fields gracefully
  • Comma-separated values — genres and countries needed exploding for accurate filtering and counting
  • Date parsing — mixed date formats in date_added required pd.to_datetime with format='mixed'
  • API design — learned to design RESTful endpoints with query params, pagination, and error responses
  • Security layers — implementing both session auth and API key auth for different use cases
  • Frontend state management — managing filters, pagination, and view state in vanilla JS

Future Improvements

  • Add a real database (PostgreSQL) for persistent data and user management
  • Implement content-based recommendation engine using NLP on descriptions
  • Add more chart types: heatmaps, treemaps, word clouds
